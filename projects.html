🎤 Qwen3-TTS Enterprise Gateway (AICESS)

Period: 2025.04 ~ Present

Role: 혼자 풀스택 설계 및 구축 (Architecture, Backend, Frontend, DevOps)

배경

AICESS의 음성 상담 센터(AIG)는 대규모 AI 음성 합성 서비스가 필요했습니다. 기존 TTS 시스템은 사용자 요청에서 음성 출력까지의 시간(TTFB)이 약 2.3초로 길어서 실시간 상담에 부적합했습니다. 또한 동시에 처리할 수 있는 채널이 제한되어 스케일링에 문제가 있었습니다. 이를 해결하기 위해 혼자서 전체 시스템을 풀스택으로 재설계하고 구축했습니다.

구현

시스템 아키텍처: FastAPI 기반 WebSocket/REST API 게이트웨이를 중심으로, vLLM-Omni(Qwen3-TTS-1.7B) 추론 엔진, Redis 분산락 & 캐싱 레이어, React 관리자 대시보드를 구성했습니다. 문장 단위 논리 청킹으로 응답 시간을 분산시키고, WebSocket을 통한 지속적 스트리밍으로 첫 음성 출력까지의 시간을 획기적으로 단축했습니다.

성능 최적화: Pre-Normalization, CUDA Graph Warmup, 세마포어 기반 동시성 제어로 GPU 메모리 효율을 극대화했습니다. Redis 캐싱으로 동일 문장 재요청 시 0ms 응답을 구현했습니다. 또한 5가지 핵심 엔지니어링 문제(팝 노이즈, GC 누수, 세션 탈취, ITN 토크나이저, CPU 병목)를 프로덕션 환경에서 발견하고 해결했습니다.

성과 (KPI)

  • ✓ TTFB: 2292ms → 334ms (85% 단축, 6.85배 개선)
  • ✓ 동시 처리 채널: 5개 → 24개 (380% 확대)
  • ✓ 초저지연 스트리밍: 500ms 이내 첫 오디오 출력
  • ✓ 프로덕션 환경: 월 10,000+ 요청 처리, 99.5% 가용성

⚙️ Kubernetes-based AI Infrastructure (Infiniic)

Period: 2022.08 ~ 2025.04

Role: 회사 최초 K8s 기반 AI 플랫폼 설계 및 구축

배경

Infiniic는 20개 이상의 ML 모델을 운영하고 있었지만, 모두 VM 환경에서 수동으로 배포되고 있었습니다. 새로운 모델을 배포하는 데 수시간이 소요되었고, 리소스 활용률은 40% 수준으로 매우 낮았습니다. 스케일링, 자동 복구, 리소스 효율 등 프로덕션 환경에 필수적인 기능들이 부족했습니다. 회사 최초로 Kubernetes 기반 AI 플랫폼을 설계하고 구축하여 이 모든 문제를 해결할 필요가 있었습니다.

구현

인프라 설계: MetalLB를 통한 로드 밸런싱, Ingress-nginx를 이용한 라우팅, PersistentVolume과 NFS를 통한 스토리지 관리를 구성했습니다. Harbor 레지스트리로 Docker 이미지를 중앙 관리하고, ArgoCD를 통해 GitOps 기반의 선언적 배포를 구현했습니다.

CI/CD 파이프라인: GitLab에서 코드 푸시 → Jenkins에서 빌드/테스트 → ArgoCD에서 자동 배포되는 완전 자동화 파이프라인을 구축했습니다. Prometheus와 Grafana로 실시간 모니터링을 구성하고, Loki와 Kibana로 로그 수집/분석을 자동화했습니다.

성과

  • ✓ 배포시간: 수 시간 → 5분 이내 (>80% 단축)
  • ✓ 리소스 활용률: 40% → 75% 이상 (비용 절감)
  • ✓ 20개+ ML 모델 자동화 배포 (확장성 확보)
  • ✓ 서비스 가용성: 99% 이상 (자동 복구)
  • ✓ 회사 최초 K8s 플랫폼 구축 (기술 리더십)

배경

자율주행 자동차의 정확한 환경 인식은 안전의 핵심입니다. 기존의 2D 세그멘테이션은 도로, 장애물, 보행자 등을 2차원 평면에서만 인식하기 때문에, 3차원 공간에서의 정확한 거리와 높이 정보를 놓칠 수 있습니다. LiDAR(3D 포인트 클라우드)와 카메라(2D 이미지)를 결합한 센서 퓨전 기반 3D 시맨틱 세그멘테이션 모델이 필요했습니다.

구현

성과

  • ✓ 특허 2건 등록 (제1발명자, 등록번호: 10-2538225, 10-2538231)
  • ✓ 우수 논문상 수상 (국방기술학회 2023.11)

🔍 OCR Dataset Annotation Tool (Eden T&S)

Period: 2022.04 ~ 2022.08

Role: 독립 개발 (Full-Stack) - GUI 도구 설계 및 구현

배경

Eden T&S는 대규모 PDF/이미지 문서에서 표(Table) 영역을 추출하는 OCR 프로젝트를 진행 중이었습니다. 하지만 학습 데이터의 수동 주석이 병목이었습니다. 매월 수만 장의 문서를 처리해야 했는데, 표 영역을 수동으로 찾고 표시하는 작업에 많은 시간이 소요되었습니다. 이 과정을 자동화하는 주석 도구(Annotation Tool)가 필요했습니다.

구현

시스템 아키텍처: PyQt를 이용한 직관적인 GUI를 설계했습니다. 사용자가 PDF/이미지를 업로드하면 Tesseract OCR과 OpenCV를 이용한 표 탐지가 자동으로 수행되고, Vision Transformer(ViT)로 표 영역을 검증합니다. 사용자는 필요시 자동 결과를 수정할 수 있도록 했습니다.

정확도 향상: RNN 기반 텍스트 추출 및 정규화로 복잡한 표 구조도 정확하게 처리했습니다. 한국어, 영어, 혼합 문서 등 다양한 언어와 포맷을 지원하도록 확장했습니다. 자동 주석 결과의 신뢰도를 평가하여, 신뢰도가 낮은 경우만 사용자 검증을 요청하는 방식으로 효율성을 극대화했습니다.

성과

  • ✓ 자동 표 탐지율: 88% 이상 (고정확도)
  • ✓ 텍스트 추출 정확도: 92% (한국어 기준)
  • ✓ 월 10,000+ 문서 자동 처리 (90% 자동화율)
  • ✓ 주석 작업 시간 85% 단축 (수작업 최소화)
  • ✓ 프로젝트 성공 시상: 사내 축하금 수령 (회사 공식 인정)
PyQt OCR (Tesseract) OpenCV Vision Transformer Python

📂 Data Collection & Preprocessing (Quhedge Internship)

Period: 2020.09 ~ 2021.09

Role: 인턴 - 데이터 엔지니어링 및 ETL 파이프라인 구축

배경

Quhedge는 금융 시장 데이터를 분석하여 투자 전략을 수립하는 회사입니다. 정확한 분석을 위해서는 신뢰할 수 있는 고품질의 데이터가 필수입니다. 다양한 소스에서 수집되는 금융 데이터들을 정제하고 통합하는 과정이 수동으로 진행되어 비효율적이었습니다. 자동화된 ETL(Extract-Transform-Load) 파이프라인 구축이 시급했습니다.

구현

데이터 수집 & 정제: 여러 금융 데이터 소스(시세 정보, 거래량, 기업 공시 등)에서 매일 50+ GB의 데이터를 수집했습니다. Python Pandas와 NumPy를 이용해 결측값 처리, 이상치 제거, 형식 정규화 등의 전처리를 수행했습니다. 데이터 결측값 처리율을 99.5% 이상으로 높여 모델 학습에 최적화된 데이터를 확보했습니다.

자동화 파이프라인: Apache Airflow를 이용해 데이터 수집 → 정제 → 검증 → 저장의 전체 과정을 자동화했습니다. SQL 기반 데이터 관리로 데이터 일관성을 보장했습니다. 데이터 품질 검증 도구를 개발하여 이상 데이터를 조기에 탐지할 수 있도록 했습니다.

성과

  • ✓ 월 50+ GB 금융 시계열 데이터 자동 수집 및 정제
  • ✓ 데이터 결측값 처리율: 99.5% 이상 (업계 수준)
  • ✓ ETL 파이프라인 자동화로 수작업 70% 감소
  • ✓ 데이터 품질 검증 도구로 오류 조기 탐지 (비용 절감)
  • ✓ 데이터 엔지니어링 기초 역량 확보 (커리어 성장)
Python Pandas / NumPy SQL Apache Airflow Data Engineering
